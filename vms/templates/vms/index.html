Welcome

This prototype illustrates the Volunteer Management Data Space concept developed in the master's thesis. To keep the prototype focused and practical, several simplifications have been made:

  • The data model uses Person → Volunteer from Schema.org, instead of Thing → Intangible → Role → VolunteerRole hierarchy described in the thesis.
  • A single demo organization is used. The application form only asks for: Name, Password, Location, and Skills.
  • The Eclipse Data Connector (EDC) is simulated through a set of simplified API endpoints, rather than a full distributed data exchange setup as discussed in the thesis.
  • Only the Volunteer data is treated as non-standardized, while Volunteer Events and Certificates are assumed to follow common standards.

Explore the three main prototype scenarios:

  • Scenario 1: Onboard an Organization into the Data Space
  • Scenario 2: Advertise Events Across Platforms
  • Scenario 3: Request a Volunteer Certificate
